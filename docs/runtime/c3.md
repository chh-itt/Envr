# C3

## Summary

C3 runtime integration provides the **C3 compiler** `c3c` via envr-managed installs from upstream GitHub Releases (`c3lang/c3c`).

## Versions

- **Remote source**: GitHub Releases (`c3lang/c3c`)
- **Labels**: semver without leading `v` (e.g. `0.7.11`)
- **Note**: upstream commonly marks the newest release as **prerelease**; envr still includes it.

## Commands

```powershell
.\envr remote c3
.\envr install c3 0.7
.\envr use c3 0.7
.\envr which --lang c3
.\envr exec --lang c3 -- c3c --version
```

## Environment

- **Home env**: `C3_HOME`
- **PATH entries**: `C3_HOME\bin`, then `C3_HOME` (host-dependent layout)

## Shims

- **Core shim**: `c3c`

When PATH proxy is enabled, `c3c` resolves to the envr-managed selected version. When disabled, the shim passes through to the next matching `c3c` on system `PATH`.

## Settings

In `settings.toml`:

```toml
[runtime.c3]
path_proxy_enabled = true
```

## Windows notes

The upstream Windows artifact is a `.zip` archive. envr extracts it and promotes a runtime home that contains `c3c.exe` either at the root or under `bin\`.
