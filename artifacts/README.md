# Artifacts directory

This directory stores generated governance and contract review outputs that are useful during development and release review, but are not part of the public user documentation surface.

## What belongs here

Examples in this directory include:

- CLI contract diff reports
- CLI capabilities snapshots
- observed metrics snapshots
- release-readiness or health summaries
- review-note drafts derived from automation

## Intended usage

These files are maintainer-facing reference artifacts.
They may be regenerated by local scripts or CI-oriented tooling and can change frequently.

They are kept in version control only when the project intentionally wants a checked-in baseline or review snapshot.
If a file here becomes purely ephemeral, prefer moving it to CI artifacts instead of keeping it committed.

## Not user documentation

Do not link end users here from the root `README.md`.
Public docs should live under `docs/`.
