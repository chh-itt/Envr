# envr 总体架构设计

## 1. workspace 结构（建议）

```text
envr/
  Cargo.toml                    # workspace root
  crates/
    envr-error/                 # 统一错误模型
    envr-domain/                # 领域模型（语言、版本、任务、配置）
    envr-config/                # 配置加载/缓存/合并（含 .envr）
    envr-platform/              # OS 抽象、PATH/链接/文件权限
    envr-download/              # 下载、校验、解压、断点续传、并发控制
    envr-mirror/                # 镜像源注册、测速、策略
    envr-shim-core/             # shim 生成与解析逻辑（无二进制入口）
    envr-runtime-node/          # Node 语言实现
    envr-runtime-python/        # Python 语言实现
    envr-runtime-java/          # Java 语言实现
    envr-runtime-go/            # Go 语言实现
    envr-runtime-rust/          # Rust 语言实现（可后置）
    envr-runtime-php/           # PHP 语言实现（可后置）
    envr-runtime-deno/          # Deno 语言实现（可后置）
    envr-runtime-bun/           # Bun 语言实现（可后置）
    envr-core/                  # 应用服务编排层（统一入口）
    envr-cli/                   # CLI 二进制
    envr-shim/                  # shim 二进制
    envr-ui/                    # UI 组件/主题（可选）
    envr-gui/                   # GUI 二进制
    envr-test-utils/            # 测试辅助
```

## 2. 分层原则

- `domain` 只放数据结构和 trait，不依赖 GUI/CLI。
- `runtime-*` 按语言拆分，统一实现 `RuntimeProvider` trait。
- `core` 只做编排，不直接持有 UI 细节。
- `cli/gui` 都调用 `core` 服务，禁止复制业务逻辑。
- `shim` 只依赖 `config/platform/domain/shim-core`，保持轻量。

## 3. 核心 trait 设计（关键）

- `RuntimeProvider`
  - `list_installed()`
  - `list_remote(filter)`
  - `install(request, progress_cb)`
  - `uninstall(version)`
  - `set_current(version)`
  - `current()`
  - `resolve(spec)`

- `MirrorResolver`
  - `resolve_index_url(language, mode)`
  - `resolve_download_url(language, version, platform, arch)`
  - `select_best(presets, strategy)`

- `ShimService`
  - `ensure_shims(language)`
  - `remove_shims(language)`
  - `sync_global_package_shims(language, version)`

## 4. 从“未完成重构项目”保留的优秀点

- workspace crate 拆分思路。
- `wx-core` 中的 platform/mirror/download/detect 经验。
- CLI 与 GUI 共用核心库方向。
- 测试分层（unit + integration + platform tests）习惯。

## 5. 从“旧项目”迁移借鉴的优秀点

- 功能覆盖完整性（尤其多语言一致体验）。
- 下载与镜像实际可用配置经验。
- GUI 侧用户路径设计（环境中心与设置聚合）。

## 6. 应避免的问题（v3 约束）

- 禁止超级 crate：`envr-core` 不能膨胀成“又大又全”。
- 禁止跨层调用：`runtime-*` 不应依赖 `cli/gui`。
- 禁止公共 API 无节制 re-export：保持明确导出边界。
- 禁止“文档先行但无人实现”：每个设计文档必须绑定里程碑任务。
