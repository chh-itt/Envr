# envr 下载与镜像源设计

## 1. 设计目标

- 下载可靠：可恢复、可校验、可取消、可并发控制。
- 镜像可控：官方优先，国内加速可选，策略可配置。
- 对语言透明：下载引擎统一，语言只提供“索引和包规则”。

## 2. 模块拆分

- `envr-download`
  - 任务管理、队列、并发限制、断点续传、校验、解压。
- `envr-mirror`
  - 预设镜像、官方源、测速与选择策略、URL 生成。
- `runtime-*`
  - 提供语言特定的 index 解析和包命名规则。

## 3. 镜像策略模型

- `official`: 始终官方源。
- `auto`: 官方 + 预设镜像测速后自动选最快。
- `manual`: 用户指定镜像 ID。
- `offline`: 仅本地缓存/内置索引（可后续支持）。

## 4. 下载流程

1. 解析版本 -> 生成目标 URL。
2. 发起 HEAD/metadata 请求（大小、校验信息）。
3. 下载到临时目录（支持断点）。
4. SHA256 校验。
5. 解压到 `runtimes/<lang>/versions/<version>`。
6. 原子落盘 + 清理临时文件。

## 5. 核心质量要求

- 下载失败可重试（指数退避）。
- 任务状态可持久化，GUI 重启后可恢复显示。
- 统一错误码：网络、证书、校验失败、解压失败、磁盘不足。
- 可观察性：进度、速度、ETA、重试次数、镜像来源。

## 6. 与既有项目整合建议

- 复用“未完成重构项目”中的 mirror registry、下载进度与并发控制经验。
- 复用“旧项目”中的多语言镜像可用性实践。
- 将镜像配置从语言内部抽离到统一策略层，减少重复代码。
