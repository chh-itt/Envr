# envr GUI/UX 增强任务清单（tasks_gui.md）

本文档在**不破坏现有功能**与 **`refactor docs/03-gui-设计.md` 性能指标**的前提下，规划「视觉与交互升级」阶段。与根目录 `tasks.md` 的关系：

- `tasks.md`：功能与交付主线（CLI、运行时、shim、测试、发布）。
- `tasks_gui.md`：**仅针对 GUI 层**的设计系统、布局、组件、动效、无障碍与下载面板体验；实现时以 `envr-ui` + `envr-gui` 为主，必要时向 `envr-core` 提需求（例如主题持久化字段已存在则只消费）。

---

## 0. 约束与验收原则

- **功能不退化**：现有路由、设置持久化、下载逻辑、运行时操作行为保持不变。
- **性能不退化**：冷/热启动、内存峰值、FPS、CPU 指标仍以 `03-gui-设计.md` 为准；长列表通过虚拟化等手段**保证**不劣化。
- **可渐进交付**：可按 Phase 合并 PR，每 Phase 有可演示的 Before/After。
- **平台差异**：Windows / macOS / Linux 三套视觉「风味」在**同一套 Design Token** 上切换，避免三套代码分叉失控。

## 1. 设计系统（Design System）落地

### [x] GUI-001 设计 Token 单源（SSOT）

- **目标**：颜色、圆角、阴影层级（elevation）、间距、动效时长/曲线、字号阶梯全部从 `envr-ui` 的 token 读取，禁止在页面里散落魔法数。
- **交付**：`crates/envr-ui/src/theme/tokens.rs`（或等价）+ 文档注释说明各 token 用途。
- **验收**：grep 业务视图中的硬编码 `#RRGGBB` 数量趋近于 0（仅允许注释或 Story/截图）。

### [x] GUI-002 平台风味映射（Fluent / Liquid Glass / Material 3）

| 平台 | 视觉关键词 | 技术落点（实现侧） |
|------|------------|-------------------|
| Windows | Fluent、Acrylic、圆角统一、elevation 阴影 | 窗口/侧栏背景使用分层；卡片 elevation；主色 `#0078D4` |
| macOS | Liquid Glass、半透明、胶囊控件、系统节奏 | 更高模糊与透明度；控件 pill；主色 `#0A84FF` |
| Linux | Material 3、动态色、大圆角 | 从壁纸/系统取 seed 色生成调色板（失败则回退品牌蓝） |

- **依赖**：GUI-001。
- **验收**：三平台同一套路由下切换主题/模式时，观感符合上表（人工截图对比 + 设计检查清单）。

### [x] GUI-003 主题模式与 Token 联动

- **目标**：明亮 / 暗色 / 跟随系统 与 Token 同步；用户给出的基准色写入 token（见第 4 节）。
- **依赖**：GUI-001、现有设置中的外观项（若已存在则改为消费 token）。
- **验收**：切换主题无「半套旧色」残留（侧栏、卡片、浮层一致）。

## 2. 布局与间距

### [x] GUI-010 应用 shell 布局规范

- **左侧导航**：固定宽度（建议 **240px**，可 Token 化 `sidebar_width`），与主内容区有清晰分隔（边框或低 elevation）。
- **默认窗口**：建议 **1200×720**（最小尺寸约 **960×600**，防止挤压）；可缩放，**主内容区**使用 `max_width` + 合理 `padding`，避免超宽一行拉满。
- **网格**：采用 **8px 基础网格**（8/16/24/32）；内容区可选 **12 列逻辑网格**（仅用于对齐，不必上重型 CSS-grid 库）。
- **验收**：主要页面左右留白一致；无「一侧贴边、一侧空一半」的失衡感。

### [x] GUI-011 页面级版式（仪表盘 / 运行时 / 设置 / 关于）

- **目标**：每类页面统一：页标题区、工具条区、内容卡片区的纵向节奏一致。
- **验收**：设置页与运行时页标题与首屏卡片顶部对齐线一致。

---

## 3. 组件库视觉升级

### [x] GUI-020 基础组件（Button / Input / Card / Section）

- **按钮**：统一高度（建议 **36px** 主按钮、**32px** 次按钮，Token 化）；主/次/幽灵/危险变体；圆角与 hover/active 状态。
- **卡片**：白/暗色卡片背景、圆角（建议 **12px**）、内边距 **16px**、elevation 1～2 级可选。
- **输入框**：统一高度与 focus 环（满足可访问性对比度）。

### [x] GUI-021 数据展示：版本列表 / 表格

- **目标**：列表行高、分隔线、空态、加载态（骨架屏，见 GUI-040）统一。
- **验收**：「智能/精确」切换后列表区域不跳动（预留骨架占位高度）。

### [x] GUI-022 图标系统（Lucide）

- **选型**：**Lucide**（MIT），风格统一、与专业工具类产品气质匹配。
- **获取方式**：在构建或开发脚本中 **从官方发布资源拉取**所需 SVG（或预生成 sprite），**禁止**手写散落 path；仓库可存 `assets/icons/lucide/` 并注明版本与许可证。
- **映射**：侧栏、刷新、设置、下载、折叠/隐藏、警告等统一 icon name → 组件封装 `Icon::Lucide(...)`。
- **验收**：主要操作点均有图标辅助（不仅依赖文字）。

## 4. 主体与色彩（基准 Token）

**与编号任务的关系**：本节是 **规范表**，不设独立 GUI-0xx。实现对应 **[x] GUI-001**（`envr-ui` `theme::tokens::base` / `SemanticColors`）、**[x] GUI-002**（各平台主色）、**[x] GUI-003**（明暗与 Token 联动）。下方「验收」指各功能在消费语义色时保持一致，可按需要 grep 审计，不必再拆任务号。

以下作为 **明亮/暗色** 的基准面（可在 GUI-001 中命名，如 `surface.page`、`surface.card`、`text.primary`）。

| 角色 | 明亮 | 暗色 |
|------|------|------|
| 页面背景 | `#F9F9F9` | `#121212` |
| 卡片背景 | `#FFFFFF` | `#1E1E1E` |
| 主文字 | `#1E1E1E` | `#E0E0E0` |
| 次文字 | `#6C6C6C` | `#9E9E9E` |

**主色（品牌）**

- Windows：`#0078D4`
- macOS：`#0A84FF`
- Linux：动态主色（Material 3），失败回退与 Windows 一致或单独 `brand.primary` token。

**语义色（全局一致）**

- 错误 `#D32F2F`
- 成功 `#2E7D32`
- 警告 `#FBC02D`

- **验收**：错误/成功/警告在 Toast、内联提示、下载状态中一致使用。

## 5. 微交互与动效

### [x] GUI-040 动效规范

- **时长**：导航切换、展开/折叠、按钮反馈等 **约 200ms**。
- **曲线**：统一 `cubic-bezier`（如 `0.2, 0, 0, 1` 或 Material 标准 easing），避免线性「机械感」。
- **原则**：动效服务可读性，禁止大面积持续动画抢焦点。

### [x] GUI-041 骨架屏（版本列表加载）

- **目标**：远程/本地列表加载时用 **渐变闪烁骨架**占位，高度与真实行接近，避免空白跳动。
- **依赖**：GUI-021。

### [x] GUI-042 下载面板显隐动画

- **目标**：显示/隐藏使用 **淡入淡出 + 轻微位移**；**不改变主内容布局宽高**（避免截图中的整体「弹跳」感）。
- **依赖**：下载面板现有状态机。

### [x] GUI-043 下载面板视觉（浮层）

- **圆角**：**12px**。
- **材质**：半透明背景 + **背景模糊**（平台能力允许时；不可用时降级为纯色半透明）。
- **依赖**：GUI-002 平台风味。

### [x] GUI-044 长列表虚拟滚动

- **目标**：版本列表等长列表采用 **虚拟化**（仅渲染可视区），保证滚动 FPS 与内存稳定。
- **验收**：在「精确模式 + 大量版本」场景下仍满足性能文档指标。

---

## 6. 可访问性（A11y）

### [x] GUI-050 对比度与焦点

- **对比度**：正文、按钮文字与背景满足 WCAG AA（关键路径尽量 AAA）。
- **焦点环**：键盘 Tab 可见焦点；禁用仅依赖 hover 才能发现的交互。

### [x] GUI-051 语义与缩放

- **系统文本缩放**（若 iced/平台支持）：布局不崩；最小点击区域建议 **44×44px** 等效（或视觉+padding 达成）。

### [x] GUI-052 减少动效

- **尊重**系统「减少动画」设置（Windows/macOS/Linux 各自查询或统一开关），开启时缩短或关闭非必要过渡。

## 7. 下载面板专项优化（针对截图问题）

### [x] GUI-060 布局：禁止文字被挤压

- **问题**：窄宽度下「折叠」「隐藏」等按钮文字竖排或被裁切。
- **方案**：
  - 工具区使用 **横向滚动** 或 **图标+tooltip** 替代窄按钮全文；
  - 或 **图标按钮**（Lucide `PanelLeftClose` / `EyeOff` 等）+ 悬停说明；
  - 设置 `min_width` / `flex` 策略，保证主标题与主操作不断行异常。
- **验收**：窗口宽度在最小尺寸下，面板内**无竖排单字、无裁切**。

### [x] GUI-061 拖拽交互

- **长按拖拽**：在面板 **标题栏/拖拽句柄区域**长按后拖拽（避免与按钮短按冲突）；实现「按下计时阈值」区分点击与拖拽。
- **位置记忆**：持久化浮层坐标（按屏幕 DPI 与窗口客户区归一化存储，避免多显示器问题）。
- **验收**：拖拽流畅；松手后位置稳定；重启后恢复。

### [x] GUI-062 与主内容的空间关系

- **默认**：左下角停靠；**不挤压**主布局；z-index 高于内容但低于模态对话框（若有）。

## 附录：为何这样规划能「直观感受到」升级

1. **先 Token、后页面**：避免「改一个按钮全站找颜色」，升级可持续。
2. **平台风味是映射层**：同一交互结构，三套观感，维护成本可控。
3. **下载面板单独成章**：对应截图痛点（挤压、拖拽、记忆），可独立验收。
4. **虚拟滚动 + 骨架屏**：直接对准「丑」背后的「空、跳、卡」三类感知问题。
5. **Lucide + 语义色 + 动效 200ms**：用户一眼能感到「像现代桌面应用」，而不是默认控件堆砌。
